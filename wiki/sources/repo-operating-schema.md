# REPO Operating Schema

Source: `REPO.md`

This file is the authoritative operating schema for maintaining the project wiki scaffold.

## Key Points

- Raw inputs live in `raw stuff/` and should be preserved as source evidence.
- Structured implementation knowledge lives in `wiki/`.
- Processed source material may move to `ingested/` when appropriate.
- Every substantive wiki page must be represented in [[index]].
- Every meaningful action must be logged at the top of [[log]].
- Error and solution pages have stricter required fields for symptoms, root cause, validation, and prevention.
